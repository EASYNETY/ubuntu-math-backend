# Webhook Signature Verification Utility

## Overview

This utility provides secure webhook signature verification for the EvriPay payment gateway integration. It implements HMAC SHA-256 signature verification with constant-time comparison to prevent timing attacks.

## Location

`src/utils/webhookSignature.ts`

## Purpose

When EvriPay sends webhook notifications to our payment system, they include a signature in the `X-EvriPay-Signature` header. This signature is generated using HMAC SHA-256 with a shared secret key. This utility verifies that:

1. The webhook request actually came from EvriPay (not an attacker)
2. The payload wasn't tampered with during transmission
3. The webhook secret matches our configured value

## Security Features

### HMAC SHA-256
Uses cryptographic hashing to create a signature that can only be generated by someone with the secret key.

### Constant-Time Comparison
Uses `crypto.timingSafeEqual()` to compare signatures in constant time, preventing timing attacks where an attacker could guess the signature by measuring how long comparisons take.

### Buffer-Based Comparison
Converts signatures to buffers before comparison to ensure proper binary comparison and prevent encoding issues.

## Usage

### Basic Usage

```typescript
import { verifyWebhookSignature } from '../utils/webhookSignature';

// In your webhook handler
const signature = req.headers['x-evripay-signature'] as string;
const payload = JSON.stringify(req.body);
const secret = process.env.EVRIPAY_WEBHOOK_SECRET!;

const isValid = verifyWebhookSignature(payload, signature, secret);

if (!isValid) {
  return res.status(401).json({ 
    error: 'UNAUTHORIZED', 
    message: 'Invalid signature' 
  });
}

// Process webhook...
```

### In Express Middleware

```typescript
import express from 'express';
import { verifyWebhookSignature } from './utils/webhookSignature';

app.post('/api/payments/webhook', (req, res) => {
  const signature = req.headers['x-evripay-signature'] as string;
  const payload = JSON.stringify(req.body);
  const secret = process.env.EVRIPAY_WEBHOOK_SECRET!;

  if (!verifyWebhookSignature(payload, signature, secret)) {
    return res.status(401).json({ message: 'Invalid signature' });
  }

  // Webhook is authentic - process it
  // ...
});
```

## Function Signature

```typescript
function verifyWebhookSignature(
  payload: string,
  signature: string,
  secret: string
): boolean
```

### Parameters

- **payload** (string): The raw webhook payload as a string. Typically `JSON.stringify(req.body)`
- **signature** (string): The signature from the `X-EvriPay-Signature` header
- **secret** (string): The webhook secret from `EVRIPAY_WEBHOOK_SECRET` environment variable

### Returns

- **boolean**: `true` if signature is valid, `false` otherwise

## How It Works

1. **Generate Expected Signature**: Creates an HMAC SHA-256 hash of the payload using the webhook secret
2. **Convert to Buffers**: Converts both the expected and provided signatures to buffers
3. **Length Check**: Verifies both signatures have the same length
4. **Constant-Time Compare**: Uses `crypto.timingSafeEqual()` to compare the buffers in constant time
5. **Return Result**: Returns `true` if signatures match, `false` otherwise

## Security Best Practices

### ✅ DO:

- Always verify signatures before processing webhook payloads
- Store webhook secrets in environment variables, never in code
- Use HTTPS for all webhook endpoints
- Log signature verification failures for security monitoring
- Return 401 status for invalid signatures (don't process the webhook)

### ❌ DON'T:

- Don't process webhooks without signature verification
- Don't expose webhook secrets in logs or error messages
- Don't use string comparison for signatures (timing attack risk)
- Don't ignore signature verification errors
- Don't reuse signatures across different webhook payloads

## Error Handling

The function returns `false` in the following cases:

- Invalid signature format
- Signature length mismatch
- Signature doesn't match computed value
- Any internal errors during comparison

It never throws exceptions, making it safe to use in production.

## Environment Configuration

Ensure `EVRIPAY_WEBHOOK_SECRET` is set in your environment:

```bash
# .env file
EVRIPAY_WEBHOOK_SECRET=your-webhook-secret-from-evripay
```

The secret is provided by EvriPay when you configure webhook endpoints in their dashboard.

## Example Attack Scenarios Prevented

### 1. Signature Forgery
**Attack**: Attacker sends fake webhook without valid signature  
**Prevention**: HMAC SHA-256 requires secret key to generate valid signature

### 2. Payload Tampering
**Attack**: Attacker intercepts webhook and modifies payment amount  
**Prevention**: Modified payload produces different hash, signature verification fails

### 3. Timing Attack
**Attack**: Attacker measures comparison time to guess signature bytes  
**Prevention**: Constant-time comparison prevents timing information leakage

### 4. Replay Attack
**Attack**: Attacker resends old webhook with valid signature  
**Prevention**: Combined with idempotency checks in webhook handler

## Performance

- **Speed**: O(n) where n is signature length (64 characters for SHA-256 hex)
- **Memory**: Minimal - creates two 32-byte buffers for comparison
- **Overhead**: Negligible - adds <1ms to webhook processing time
